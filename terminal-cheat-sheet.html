Workflows

Start a new session and name it

  1. Press ⌘T to create a new terminal session.
  2. Press ctrl+a, then , to rename the tab/window.
  3. Enter a short name such as build or tests.

Split pane and navigate between panes

  1. Press ⌘D for a right split or ⌘⇧D for a down split.
  2. Use ⌘⌥← / ⌘⌥→ / ⌘⌥↑ / ⌘⌥↓ to focus a pane by direction.
  3. Use ctrl+a, then Shift+arrows to resize the active pane.
  4. Press ⌘⌥W to close only the focused pane.

Detach from a session and reattach

  1. Press ctrl+a, then d to detach the client while the session keeps running.
  2. Open Harness again or return to the session list.
  3. Use ⌘1 ... ⌘9 or ⌘[ / ⌘] to reselect the session.

Copy text in copy mode

  1. Press ctrl+a, then [ to enter copy mode.
  2. Move with h / j / k / l, PageUp, or PageDown.
  3. Press v, V, or C-v to begin a selection.
  4. Press y or Enter to yank to clipboard and exit copy mode.

Open and use the command palette

  1. Press ⌘; to open the command prompt.
  2. Use commands such as :split README.md, :vsplit README.md, or :grep TerminalHostView.
  3. Use :board to open the session board or :attention to jump to a session needing attention.

Find a file and inspect project text

  1. Press ⌘P to find a file, or use ctrl+t to paste a fuzzy-picked path at the shell prompt.
  2. Run rg "pattern" to search text across the project.
  3. Use rg -l "pattern" when you only need matching filenames.
  4. Pipe file candidates through fd -t f | fzf when the result set is large.
