- [2026-07-08] **Terminal ASCII-QR "too big" complaint: font size / module count shrink the true on-screen footprint; ASCII-packing scheme (half-block vs quadrant-block) does not.** `qrAsciiArt` (`MobileBridgeServer.swift`) went double-width (78x39 chars, confirmed too big) ‚Üí half-block ‚ñÄ‚ñÑ‚ñà (39x17, correctly square ‚Äî 2 module-rows packed per terminal row, matches terminal chars rendering ~2x taller than wide) ‚Üí quadrant-block (20x17, packs a 2x2 module block per char) ‚Äî this last one was a regression: compressing 2 module-*columns* per char on top of the half-block's existing 2-module-*row* compression, with no compensation for the terminal's ~2:1 char height:width ratio, stretched the QR visibly non-square (caught via user screenshot, not caught by any automated check).
